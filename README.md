# AutoBot v2

Telegram-бот на Aiogram 3 для автомобильной диагностики, гаража, поиска сервисов и истории обращений.

## Что умеет

- Принимает описание проблемы с автомобилем.
- Уточняет автомобиль по VIN или вручную.
- Даёт первичный совет и диагноз через OpenAI.
- Подбирает запчасти по авто, вероятной неисправности и бюджету через mock-каталог.
- Анализирует фото проблемы через OpenAI Vision с уточнением типа снимка.
- Блокирует самостоятельный ремонт для опасных симптомов.
- Ищет сервисы, шиномонтаж, АКБ, заправки и эвакуатор рядом через Google Places и Google Maps.
- Хранит гараж автомобилей.
- Хранит историю обращений.
- Хранит напоминания по обслуживанию и сам присылает уведомления по датам.
- Хранит FSM-состояния в SQLite, поэтому не теряет текущий шаг после перезапуска.
- Использует локальные ответы и кэш OpenAI для частых проблем.
- Запоминает, помог ли ответ пользователю, и отключает слабый кэш.

## Быстрый запуск

```bash
python3 -m venv .venv
source .venv/bin/activate
python -m pip install -r requirements.txt
cp .env.example .env
```

Заполните `.env`:

```bash
TELEGRAM_TOKEN=your_telegram_bot_token
OPENAI_API_KEY=your_openai_api_key
GOOGLE_PLACES_API_KEY=your_google_places_api_key
```

Запуск:

```bash
python bot.py
```

## Запуск через Docker

```bash
cp .env.example .env
```

Заполните `.env` реальными ключами, затем:

```bash
docker compose up -d --build
```

Логи:

```bash
docker compose logs -f autobot
```

Остановка:

```bash
docker compose down
```

SQLite-база хранится в папке `data/`, поэтому не пропадает при пересборке контейнера.

## Backup SQLite

Локально:

```bash
python scripts/backup_sqlite.py
```

В Docker:

```bash
docker compose exec autobot python scripts/backup_sqlite.py
```

Копии сохраняются в `backups/`.

## Минимальный деплой на VPS

1. Создайте Ubuntu VPS.
2. Установите Docker и Docker Compose.
3. Скопируйте проект на сервер.
4. Создайте `.env` на сервере и заполните реальные ключи.
5. Запустите:

```bash
docker compose up -d --build
```

6. Проверьте:

```bash
docker compose logs -f autobot
```

Если сервер перезагрузится, контейнер поднимется автоматически благодаря `restart: unless-stopped`.

## Проверка интеграций

Скрипт проверяет реальные Telegram, OpenAI и Google API, но не печатает ключи. Он делает реальные API-запросы, поэтому запускайте его только когда хотите проверить живые ключи:

```bash
python scripts/check_integrations.py
```

Ожидаемый безопасный вывод:

```text
TELEGRAM_TOKEN OK
OPENAI_API_KEY OK
GOOGLE_PLACES_API_KEY OK
Telegram getMe OK
OpenAI OK
Google Geocode OK
Google Places OK
```

## Команды

- `/start` — главное меню.
- `/help` — что умеет бот.
- `/cancel` — сбросить текущий сценарий.
- `/reset_me` — полностью очистить ваши тестовые данные в боте.
- `/garage` — открыть гараж.
- `/history` — последние обращения.
- `/photo` — анализ фото.
- `/reminders` — активные напоминания.
- `/add_reminder` — добавить напоминание.
- `/done_reminder ID` — закрыть напоминание.
- `/admin` — закрытая админка, доступна только ID из `ADMIN_IDS`.
- `/admin_stats` — сводка по пользователям, обращениям, кэшу и API.
- `/admin_users` — последние пользователи.
- `/admin_problems` — частые проблемы.
- `/admin_bad_answers` — ответы, которые получили плохие оценки.
- `/admin_errors` — последние API-ошибки.
- `/test_mode` — включить/выключить тестовый режим для администратора.
- `/stats` — API-статистика и примерная стоимость. Если `ADMIN_IDS` заполнен, команда доступна только администраторам.
- `/errors` — последние API-ошибки для администратора.

## База данных

По умолчанию используется SQLite-файл `autobot.db`.

Основные таблицы:

- `users` — пользователи.
- `cars` — автомобили пользователя.
- `issues` — история обращений.
- `ai_response_cache` — сохранённые ответы OpenAI.
- `ai_response_feedback` — оценки ответов пользователями.
- `api_usage` — статистика OpenAI/Google, токены, попытки, ошибки и примерная стоимость.
- `reminders` — напоминания пользователя.
- `fsm_storage` — текущие шаги диалогов Aiogram.

## Тестовый режим

Администратор может включить тестовый режим:

```bash
/test_mode
```

После этого при поиске сервисов бот показывает служебные детали: выбранную категорию, радиус, минимальный рейтинг, лимит выдачи, координаты и количество найденных мест.

## Поиск сервисов

После выдачи сервисов можно уточнять поиск кнопками:

- `➕ Показать ещё` — расширить выдачу.
- `📍 Искать ближе` — уменьшить радиус.
- `⭐ Рейтинг ниже` — снизить минимальный рейтинг.
- `📡 Увеличить радиус` — если ничего не найдено.
- `🔁 Поменять категорию` — выбрать эвакуатор, СТО, шиномонтаж, запчасти и т.д.
- `✏️ Другой адрес` — ввести новую локацию.

Далёкие результаты отсекаются по расстоянию от координат пользователя.

## Расширенный гараж

Авто можно добавлять простым форматом:

```text
Toyota Camry 2019
```

Или расширенным форматом:

```text
Toyota Camry 2019 | 120000 км | 2.5 | автомат | бензин | Камри
```

Расширенные данные передаются в диагностику, чтобы советы были точнее.

## Подбор запчастей

Пользователь может нажать `🔩 Найти запчасть` или перейти к подбору после диагностики, если бот уверен, что нужна замена детали.

Сценарий:

1. выбран автомобиль из гаража или введён вручную;
2. указана деталь;
3. выбран бюджет;
4. бот показывает OEM, качественный аналог, дешёвый аналог и поисковые ссылки.

Mock-каталог уже содержит тестовые данные для:

- Toyota Sequoia 2018;
- Toyota Camry 70;
- Hyundai Elantra;
- Kia Sportage;
- BMW G30.

Классификация вариантов:

- `original_oem` — оригинал;
- `trusted_aftermarket` — качественный аналог;
- `cheap_aftermarket` — дешёвый аналог;
- `unknown` — неизвестный вариант.

Для тормозов, рулевого, подвески и топливной системы бот не рекомендует `unknown` как основной вариант. Если оригинал не проходит по бюджету, основным вариантом становится `trusted_aftermarket`. Если выбран дешёвый аналог, бот предупреждает о возможном снижении ресурса.

Пока платные API не используются. Вместо этого бот создаёт готовые поисковые ссылки Google по OEM-номеру, бренду и русскому запросу для Казахстана.

## Ручной ввод адреса

Чтобы Google не уводил поиск в другой город или страну, ручной адрес вводится в два шага:

1. город;
2. улица, район или точный адрес.

По умолчанию geocode ограничен Казахстаном:

```bash
DEFAULT_COUNTRY=Kazakhstan
DEFAULT_COUNTRY_CODE=KZ
```

Если бот будет работать в другой стране, поменяйте эти значения в `.env`.

## Как работает экономия OpenAI

Для частых проблем бот сначала проверяет локальные готовые ответы. Если подходящего ответа нет, он проверяет кэш в базе. Только после этого делает запрос в OpenAI и сохраняет успешный ответ в `ai_response_cache`.

После ответа бот может спросить, помог ли совет. Если сохранённый ответ несколько раз получает оценку "не помогло", бот перестаёт использовать этот кэш и снова обращается к OpenAI.

## Напоминания

Добавление:

```bash
/add_reminder
```

Формат:

```text
Замена масла | 2026-08-01 | 120000 км
```

Повторяющееся напоминание:

```text
Замена масла | 2026-08-01 | 120000 км | каждые 8000 км | каждые 180 дней
```

Когда такое напоминание закрывается через `/done_reminder ID`, бот создаёт следующее с новой датой и/или пробегом.

Бот проверяет напоминания фоном. Интервал проверки настраивается:

```bash
REMINDER_CHECK_INTERVAL_SECONDS=3600
```

## API-статистика и стоимость

Бот записывает вызовы OpenAI и Google в таблицу `api_usage`. Для OpenAI сохраняются токены, количество попыток, успех/ошибка и примерная стоимость. Для Google сохраняются обращения к nearby search, place details и geocode.

Стоимость задаётся через `.env`, потому что тарифы провайдеров могут меняться:

```bash
OPENAI_INPUT_COST_PER_1M=0
OPENAI_OUTPUT_COST_PER_1M=0
GOOGLE_PLACES_COST_PER_REQUEST=0
GOOGLE_GEOCODE_COST_PER_REQUEST=0
```

Если оставить значения `0`, статистика вызовов и токенов будет работать, а стоимость будет считаться как `$0.000000`.
